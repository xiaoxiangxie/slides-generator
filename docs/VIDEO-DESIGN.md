# S01-Neon-Cyber 项目经验教训总结

## 正确流程（一步到位）

### 第一步：确定视频总时长
问自己：这个内容正常人讲话需要多久？
- 1082 字 ÷ 5字/秒 = 216s 口播
- 44 条字幕 × 1.2s 气口 = 52.8s
- **总时长 ≈ 270s**（取 270 的倍数）

### 第二步：生成字幕时间轴
```
每条字幕时长 = 字数 ÷ 5（5字/秒是正常语速）
相邻间隔 = 1.0 ~ 1.5s（气口）
最后结束时间 ≤ 视频总长 - 1s
```

### 第三步：验证可行性
生成前先算：
```
内容总时长 + 气口总时长 = 是否 ≤ 视频总长？
```
如果超出，**调整视频时长**而不是压缩字幕。

### 第四步：设计场景时长
口播结束后 + 1.5s 缓冲 = 场景时长。
按比例分配到各场景（保持原有比例）。

### 第五步：渲染 + 验证

---

## 设计审查清单（每次）

- [ ] 口播总时长是否 ≤ 视频总长？（先算清楚再动手）
- [ ] 相邻字幕之间是否 ≥ 1s 气口？
- [ ] 每条字幕的时长是否在 2.5s ~ 8s 之间？（太短赶，太长拖）
- [ ] 最后一条字幕是否在视频结束前 1s 以上完成？
- [ ] 场景时长 = 最后一条字幕时间 + 1.5s 缓冲？

---

## 核心数字

| 参数 | 数值 | 说明 |
|------|------|------|
| 正常语速 | 5 字/秒 | 剪映自动朗读速度 |
| 最短字幕 | 2.5s | 约 12 字 |
| 最长字幕 | 8s | 约 40 字 |
| 最小气口 | 1.0s | 相邻字幕间隔 |
| 推荐气口 | 1.2s | 舒适，不拖 |
| 场景缓冲 | 1.5s | 最后一条字幕后留白 |

---

## 快速计算公式

```
视频总长 = 内容字数 ÷ 5 + (字幕条数 - 1) × 气口 + 1.5s
```

例：1082 字，44 条，1.2s 气口
= 1082/5 + 43×1.2 + 1.5
= 216.4 + 51.6 + 1.5
= **269.5s → 取 270s (8100帧)**
